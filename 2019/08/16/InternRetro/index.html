实习总结

实习总结？实习总结是什么？ 谁提的需求？ 表面需求背后的真正需求是什么？
这个大概是我来到thoughtworks之后的一个收获。

要做什么？为什么要这么做？这么做有什么意义？···要成为一个合格的程序员，能够依葫芦画瓢地写出合格的业务代码就可以了。But if you want to be a Developer, you need to develop yourself：清晰的思维，专业的技能，明确需求，规划方案，不断尝试与调整，实现目标。
感谢组内的BA同学，在反馈会议上对主题与流程的把控，让我看到了一次精彩的问题梳理与解决的过程。

那么，关于实习总结…

  • 实习总结是什么？
    由人事部门布置给公司实习生，在实习结束之前由实习生，即我本人，完成；通过某种文本形式反馈给部门负责人。

  • 实习总结有什么用？
    对我来说，这是一个对两个多月以来逐渐熟悉，新鲜感逐渐减少的过程的总结反馈：用于回顾过去的收获与不足；及时提醒自己保持新鲜感，不要对熟悉的事物视而不见；以及，珍惜最后几天在公司的实习生活，完成在最后几天该做的事情。感性地讲，或许也是一种仪式感吧。
    （按照我的理解）对公司来说，实习生的总结也是公司反馈文化的一部分。正如我似乎能在项目上测出一些其他人没找到的bug一样，从新人的视角观察公司，或许总能看到一些由于熟悉而不经意忽略的地方。

  • 那么，实习总结该如何写？
    每个人风格各异，有人喜欢用时间线遍历自己在公司的日常，有人喜欢用思维导图画出自己接触到的知识点，有人喜欢以人为本，探讨自己从每位同事身上学到了什么…我的话，可能是个套娃meta爱好者，写实习总结一定会写成关于实习总结的总结，还喜欢打破第四面墙。至于将来某天发愁实习总结怎么写的你从某个链接中扒出这篇总结看到这里的你，我会说，开心就好。

那你到底在这学了个啥？

我学了什么？两个月之前，我对Java一无所知，在学校实验室里一个人痛苦摸索，不停地咒骂自己发愁找不到工作担心无法毕业对不起老师栽培。
然后，我在公司得到了同事的帮助与讲解，避开了若干坑，快速地上手了Java和Spring的基础知识，基本入门git的操作。
我在公司里收获了同事的善意，体会到了友好的氛围以及他们对自己生活的满足感；
我在项目中找到自己的角色，做出工作,得到认可。
我在公司见到了专业性。了解了敏捷开发、测试驱动、持续交付等概念，明白了自己当初看论文写代码时的低效与混乱。
在每天的站会上反思自己的每天的进步与不足，思考自己对项目的贡献；
在每天的Code Review上将自己的弱鸡代码展示出来供大家批判，收获了代码风格的规范，写法的优化与多种实现，学到了各种不知道的知识点。
了解了关于进度把控，任务分解，估点等对于一个项目进展的知识，可以尝试用到自我管理与科研项目管理的任务上。
或者你想听Java知识点？那移步本人github内Java-Project-Review应该会慢慢填坑。
(或许还能收获留在公司工作的机会。)
学会了“Coffee in，Code out”，(git push/pull好爽啊，我也要成为地铁站！)

在公司，我发现了自己将来的不同可能，虽然对将来的路还有迷茫与犹豫，但是能发现其中几条路的践行者，对我（不存在）的人生规划意义重大。祝福他们在自己的道路上越走越好。

还有什么要反馈的吗？

感谢zhiheng的入门帮助，让我在最短时间内理清思路，掌握知识，明确自己该做什么。
感谢组内同事的帮助让我成长，以及他们的存在让我看到人生的各种可能性。（以及给我犯下的错误善后，改了一上午数据库真的辛苦了。）

存在的问题的话，公司的咖啡好像不怎么好喝的样子，很少有人愿意喝···建议改善，可以接受付费；还有就是Mac太好用了实习生没资格回购

暂时没了吧，持续集成持续交付，假如上传到github这种形式没问题的话，这篇总结还可以之后慢慢改。(虽然hackMD比github写markdown好用多了..)
